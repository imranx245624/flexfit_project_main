1.1 Background:  
 
Physical fitness and guided strength training have become essential for health 
and well-being . While many people wish to exercise correctly, a significant number of 
beginners and home users lack access to qualified trainers. This gap leads to improper 
technique, reduced exercise effectiveness, and potential injury. Traditional solutions 
include in -person trainers, expensive motion -capture hardware (e.g., Kinect, Vicon), or 
wearable sensors that impose cost and accessibility barriers.  
 
Concurrently, modern web technologies and machine learning make it possible 
to estimate human pose using a plain webcam inside a browser. Browser -based pose 
estimation (TensorFlow.js, MediaPipe) coupled with WebGL (Three.js) for 3D 
visualization can provide low -cost, accessible real -time guidance. FlexFit leverages 
these technologies  to provide users with exercise demonstrations, live posture 
monitoring, instant voice feedback, and 3D muscle visualization — all within a Web 
Application  that works across desktop and mobile browsers.  
 
Existing fitness apps (video/ tutorial -based ) offer recorded lessons but lack real -
time correction. Research prototypes using depth cameras or server -side models show 
good accuracy but are not practical for general users. FlexFit aims to bridge the gap by 
combining client -side pose estimation, immed iate corrective voice guidance, and 
engaging visualization to simulate a trainer experience at low cost.

2.1 Existing System  
Many commercial fitness platforms (e.g., Nike Training Club, Freeletics, 
YouTube fitness channels) provide high -quality exercise videos, guided programs, and 
progress tracking. However, they generally lack live correction. Research systems have 
used depth sensors (Microsoft Kinect) or wearables to estimate pose more accurately; 
these solutions perform well but require additional hardware, limiting adoption.  
 
Academic and open -source pose estimation frameworks include OpenPose, 
BlazePose (MediaPipe), and MoveNet (TensorFlow.js). OpenPose offers detailed 
multi -person body pose estimation , but is heavy and usually server/GPU dependent. 
BlazePose is optimized for mobile and offers good accuracy. MoveNet (TensorFlow.js) 
is optimized for real -time in -browser use. Each approach has tradeoffs between 
accuracy, latency, and deployment complexity.  
 
Shortcomings in existing systems:  
• Hardware dependency (depth cameras or wearables).  
• Server -side processing or GPU requirement (costly).  
• Lack of integrated voice correction + 3D visual explanation.  
• Limited focus on web -first.  
 
 
8 
 2.2 Proposed System:  
 
FlexFit is proposed as a  browser   first Web App  that performs most pose processing 
client -side (in the browser) using TensorFlow.js MoveNet (or MediaPipe , where 
appropriate). The architecture includes:  
 
• Frontend:  UI components (VideoScroller, StartWorkout, AIWorkout), routing, 
responsive design, Lottie/Three.js integrations.  
 
• Client Pose Engine:  In-browser pose detection (MoveNet) that outputs 
keypoints and joint angles.  
 
• Feedback Engine:  Computes corrective heuristics (angle thresholds, repetition 
counts) and triggers voice prompts via Web Speech API.  
 
• 3D Visualization Module:  Three.js to render a simplified human model and 
highlight engaged muscles.  
 
• Backend (planned):  Spring Boot REST API to store user profiles, session logs 
in PostgreSQL; Redis for leaderboard caching and session fast reads.  
 
• Persistence & Reporting:  Session summaries stored in DB and PDF reports 
generated on demand

 4.3.3 Algorithms Design:  
The FlexFit project relies on algorithms to process pose data, calculate workout 
performance, and provide real -time corrective feedback. These algorithms ensure 
accuracy, efficiency, and a near -instantaneous response to the user’s actions.  
 
1. Pose Detection Algorithm  
• Technology: TensorFlow.js , MoveNet or MediaPipe BlazePose.  
• Input: Webcam video frames.  
• Process:  
o Capture a video frame.  
o Extract 17 –33 body keypoints (depending on model).  
o Store coordinates + confidence scores.  
• Output: Pose keypoints for further analysis.  
 
2. Repetition Counting Algorithm  
• Uses joint angle thresholds (e.g., elbow angle for push -ups).  
• Example logic for Push -ups: 
o If elbow angle < 70° → "Down" position.  
o If elbow angle > 160° after "Down" → Count +1 repetition.  
• Ensures reps are only counted when a full motion cycle is complete.  
 
3. Accuracy Calculation Algorithm  
• Compares real -time user pose with predefined “ideal pose” dataset.  
• Formula:  
• accuracy = (Σ (1 - |θuser - θideal| / θideal) / n) * 100  
o θ = joint angles  
o n = number of joints considered  
• Produces an accuracy score (%) shown after each workout.  
 
 
 
 
 
37 
 4. Voice Feedback Algorithm  
• Rules -based approach using Web Speech API . 
• Example:  
o If back angle > ideal by 15° → Voice feedback: “Keep your back 
straight.”  
o If reps counted = milestone (e.g., 10 reps) → Voice feedback: “Great 
job! Keep going.”  
 
5. Leaderboard Scoring Algorithm  
• Rank score = weighted formula:  
• rankScore = (accuracy * 0.6) + (reps * 0.3) + (duration * 0.1)  
• Cached in Redis for faster leaderboard updates.  
 
6. Report Generation Algorithm  
• Session data retrieved from DB.  
• Aggregates: total sessions, best accuracy, average reps.  
• Converts into a PDF report using jsPDF/PDFBox.

 4.5 Security Issues:  
 
 Data Protection:  
• End-to-end encryption for all user data in transit (TLS 1.3)  
• Password hashing using bcrypt (work factor: 12)  
• JWT tokens with limited lifespan (15 minutes access, 7 days refresh)  
• Secure HTTP headers (CSP, XSS protection, HSTS)  
 
Privacy Considerations:  
• Video processing occurs entirely client -side 
• No raw video data transmitted to servers  
• Only anonymized pose coordinate data stored  
• GDPR -compliant data retention policies  
 
Authentication Security:  
• Rate limiting on login attempts (5 attempts/hour)  
• Automatic session expiration after 24 hours of inactivity  
• Secure cookie attributes (HttpOnly, Secure, SameSite)  
• OAuth 2.0-ready  architecture for future social login integration
